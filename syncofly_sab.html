① Orbit Scenario
Source A sits at center. All agents orbit around it at a radius set by their current inter-onset-interval. At every predicted beat onset, each bot takes an angular step. Social coupling coordinates direction only — agents that overlap pass through each other and exchange direction; the higher-confidence agent's direction wins.
② Bottleneck Exposure
All agents start frozen in a line. Only B1 is within hearing range of source A. Once B1 entrains and locks the pattern into memory, it marches toward B2 and wakes it up by coupling. Entrainment propagates one agent at a time until B5 reaches goal C.

SYNCOFLY — User Guide

Browser-based multi-agent collective rhythmic resilience platform

① What This Does

SYNCOFLY tests how a group of five agents coordinates around a shared rhythm and how well it recovers when disrupted. Source A emits a repeating pattern. Agents relay and match it to unlock target C. Each agent can carry the Grid Model (M4p hierarchical meter-predictive oscillator) or operate reactively.

② Setup Tabs

① PatternClick the polar clock to design the rhythmic pattern. Outer = binary (16th notes), inner = ternary (24th notes). Pattern B is also configured here if using a pattern switch.
② AgentsToggle the Grid Model on/off per agent. Without it, agents are fully reactive followers. Also mark noisy agents here.
③ PerturbationSelect perturbation type: None, Beat Omission (randomly dropped beats during shock phase), or Pattern Switch (A→B→A sequence). Shock/recovery and baseline durations are set here too.

③ Visuals

  • A = emitter (source). Pulses on each beat. Dots show current beat within cycle.
  • B = lock (target). Beat-by-beat match indicators below it. Flashes and shows ✓ when group relays correct pattern.
  • Phase arc on each agent = internal metrical phase (0→1 per cycle). Agents in sync have aligned arcs.
  • Confidence halo = glow brightness reflects M4p model confidence.
  • Red ring = noisy agent. Red border = shock phase active.

④ Metrics

T₁ (first sync)Time from onset until first correct sync per agent.
T_groupTime until all agents synchronize to some rhythm.
A_syncGroup accuracy at T_group — proportion locked to correct pattern.
Retention rate (R)Proportion of agents that maintained correct pattern through end of shock.
Corruption typeFor each corrupted agent: "Pattern sub." = locked to wrong pattern (high conf); "Sync loss" = lost stable entrainment (low conf).
T_rec (per-agent)Time from shock onset until each corrupted agent re-achieved correct sync.
T_full (group)Time from shock onset until ALL corrupted agents recovered. Primary group resilience measure.
Recovery rate (RR)Proportion of corrupted agents that recovered within the observation window.
Phase varianceVariance in internal phase across agents. Low = synchronized group.
